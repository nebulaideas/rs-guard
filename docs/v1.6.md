# v1.6.0 — Review depth

**Status:** Planned  
**Milestone:** [v1.6.0 - Review depth](https://github.com/nebulaideas/rs-guard/milestone/2)  
**Theme:** Security and coverage foundations for the core review pipeline — without multi-pass scale (that is v1.8).

This release focuses on what you can review safely and how tooling consumes the result: outbound secret hygiene, configurable size/path gates, machine-readable output, local branch diffs, and doc accuracy.

---

## Goals

1. Never send obvious secrets in diffs to third-party LLMs by default.
2. Review more real PRs by raising/configuring hard limits and excluding noise paths.
3. Let CI and scripts consume a structured JSON result without scraping terminal text.
4. Match "PR-like" local reviews via `base...HEAD`, not only staged changes.
5. Align architecture/performance docs with the post-v1.5 implementation.

## Suggested implementation order

```text
#106 (docs drift)          -- can land anytime / first
#102 (outbound redaction)  -- security foundation
#103 (size limits + paths) -- coverage foundation
#105 (local branch-diff)   -- local DX
#104 (--format json)       -- tooling integration
#107 (CriticalBugs alias)  -- small cleanup (pair with #106 if desired)
```

**Rationale:** Documentation can ship independently. Secret redaction should land before broader adoption of larger diffs. Path filters (#103) reduce cost before users raise limits. JSON output is independent of git-diff source.
