# v1.8.0 — Scale

**Status:** Planned  
**Milestone:** [v1.8.0 - Scale](https://github.com/nebulaideas/rs-guard/milestone/4)  
**Theme:** Handle monorepos and large PRs, better metrics, more provider presets, and broader release artifacts.

Assumes v1.6 path/size controls and preferably v1.7 structured findings for multi-pass merge quality.

---

## Goals

1. **Multi-pass** review for diffs that cannot fit a single context window — opt-in, cost-bounded.
2. **`.rs-guardignore`** and language-aware prompt selection for monorepo DX.
3. **Real API token usage** in metrics when providers return `usage`.
4. **Ollama + Gemini** first-class presets (OpenAI-compatible path).
5. **Multi-arch** release binaries and optional binary size CI budget.

## Design constraints

- **Multi-pass is opt-in** — default CI must not explode cost or wall-clock time.
- **Cache keys per chunk** must include chunk identity + same parameters as single-pass.
- **Ollama** is local-oriented; document that CI still needs a remote provider unless self-hosted runners expose Ollama.
- **Native Anthropic Messages API** (non-OpenAI-compatible) is **out of scope** unless opened as a follow-up; prefer OpenRouter path in docs.
