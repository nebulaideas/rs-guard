# v1.7.0 — GitHub-native UX

**Status:** Planned  
**Milestone:** [v1.7.0 - GitHub-native UX](https://github.com/nebulaideas/rs-guard/milestone/3)  
**Theme:** Make rs-guard feel like a first-class GitHub review bot — inline comments, branch-protection Check Runs, robust structured verdicts, and resilient review submission.

Builds on v1.6 foundations (especially larger/cleaner diffs and JSON output). Prefer landing v1.6 security/size work first where dependencies exist; Check Runs and body truncation can proceed independently.

---

## Goals

1. Post findings as **inline review comments** (path + line) when the model provides them.
2. Optionally publish a **Check Run** so branch protection does not depend on `APPROVE` / `REQUEST_CHANGES` permissions.
3. Prefer **structured JSON verdicts** (with text metadata fallback) to reduce truncation/parser drift.
4. **Never fail** a useful review solely because the body exceeds GitHub's 65k character limit.

## Permissions note

- Review events: `pull-requests: write` (existing).
- Check Runs: `checks: write` (document in bot setup).
- `GITHUB_TOKEN` often cannot `APPROVE` / `REQUEST_CHANGES` — Check Runs + COMMENT fallback remain important.
